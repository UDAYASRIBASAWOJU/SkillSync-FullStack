1. The Authentication & Routing Gate

How SkillSync locks down its DOM trees using React Router and Redux Toolkit. This flow maps the exact journey from the Google Login button to securing protected dashboard routes.

Step 1: Google OAuth Call

The user clicks 'Sign in with Google'. The @react-oauth/google provider catches the pop-up success and extracts the Google `credential` string.

const loginMutation = useMutation({
  mutationFn: (token) => axios.post('/api/auth/google', { token }),
  onSuccess: (data) => dispatch(setCredentials(data))
});

Step 2: Redux Storage (authSlice)

The successful backend response yields our signed JWT and User object. The Redux Toolkit authSlice synchronously updates global UI memory. Crucially, redux-persist immediately serializes this into local browser storage.

State Mutation isAuthenticated → TRUE
user Role → 'LEARNER' | 'MENTOR' | 'ADMIN'

Step 3: Gateway Resolution (React Router)

The <ProtectedRoute> wrapper guarding the `/dashboard` route intercepts the render phase. It reads `useSelector(state => state.auth)`. Seeing validation, it drops the shield and mounts the requested UI components.

if (!isAuthenticated) return <Navigate to="/login" replace />;
if (requiredRole && userRole !== requiredRole) return <Navigate to="/unauthorized" />;
return <Outlet />; // Render children securely

2. The Network Interceptor Engine

How the frontend ensures absolutely every single outbound API request is aggressively appended with cryptographic proof of identity.

The Outbound Hook (Axios Request)

A component calls a React Query hook (e.g. fetchMentors()). Before the physical HTTP request leaves the browser tab, the centralized Axios instance literally catches it mid-flight.

Header Injection

The interceptor accesses the Redux Store, extracts the current active JWT accessToken, and mutates the outbound HTTP headers.

api.interceptors.request.use((config) => {
  const token = store.getState().auth.token;
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

Inbound Fallback (401 Trapping)

If the backend rejects the token as "Expired" (401 Unauthorized), the response interceptor catches the error BEFORE it reaches the UI. It pauses the request queue, hits /api/auth/refresh, acquires a new token, updates Redux, and replays the failed request flawlessly without the user ever noticing.

3. Payment Overlay & Booking Workflow

The exact UI workflow allowing a Learner to lock a Mentor's time slot and execute a Razorpay financial settlement through the Frontend.

Step 1: The Schedule Grid

The frontend calls the Session Service to gather the Mentor's `available_slots`. The Learner clicks a slot, executing a React Query Mutation to lock it.

Step 2: Backend Initialization

Instead of confirming the booking, the backend fires to the Payment service and replies to the Frontend with a generated razorpay_order_id. The booking is now mathematically `PENDING`.

Step 3: Razorpay Hijack

The frontend injects the official Razorpay JS SDK script into the DOM. It mounts an iframe overlay directly on the screen, forcing the user into the financial gateway without rerouting the URL.

const options = {
  key: import.meta.env.VITE_RAZORPAY_KEY,
  order_id: backendResponse.orderId,
  handler: async function (response) {
    await verifyPaymentCallback(response);
  }
};
const rzp = new window.Razorpay(options);
rzp.open();

Step 4: Callback & Resolution

Once Razorpay succeeds, the frontend captures the payload and Posts it back to our API Gateway to officially notify the Session Service that the PENDING state is cleared and finalized.

4. STOMP Realtime Chat Engine

How React hooks into the Notification Service's active TCP streams to power instantaneous Group Chat messages without utilizing standard REST fetches.

Mount & Handshake

When a user navigates to `/dashboard/chat`, the ChatContext mounts. It fires up SockJS and upgrades the standard HTTP connection to a continuous WebSocket connection.

const socket = new SockJS('https://api.../ws');
const stompClient = Stomp.over(socket);
stompClient.connect({ Authorization: `Bearer ${token}` }, onConnect);

Topic Subscription

React dynamically loops through the User's groups array and actively commands the STOMP client to "listen" to those explicit backend topic channels.

stompClient.subscribe(`/topic/groups/${groupId}`, (payload) => {
  const newMessage = JSON.parse(payload.body);
  setMessages(prev => [...prev, newMessage]); // React State Update
});

DOM Painting

When another user chats via the Notification Server Broker, the frontend setMessages executes instantly. The React DOM reconciler diffs the state and physically drops the new HTML chat bubble on the screen within ~20ms.

5. Administrator Deep Control

How the frontend scales administrative power to manage pending Mentor approvals across the system gracefully.

Paginated Server State

React Query fetches /admin/mentors/pending?page=0&size=10. This guarantees the browser isn't overloaded rendering 5,000 pending applications at once. The UI maps the `content` array into Data Table rows.

Optimistic UI Updates

When the Admin clicks 'Approve', a TanStack Mutation triggers. Instead of waiting for the User Service to reply (which creates UI lag), the frontend optimistically edits the React Query Cache immediately, vanishing the table row instantly to feel infinitely fast, falling back only if the server returns an explicit 500 error.

onMutate: async (id) => {
  await queryClient.cancelQueries(['pendingMentors']);
  const previous = queryClient.getQueryData(['pendingMentors']);
  queryClient.setQueryData(['pendingMentors'], old => old.filter(m => m.id !== id));
  return { previous };
}
