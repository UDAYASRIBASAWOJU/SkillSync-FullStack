1. The CI/CD Automation Matrix

When an engineer targets pushing to `main`, GitHub Actions spins up an ephemeral runner to physically execute the entire application build natively before dispatching it to production.

Phase 1: Environment Spin-up & Extraction

GitHub executes ubuntu-latest. The actions/checkout@v4 action hooks into our repo and literally clones the codebase into the virtual machine. It then installs Java 21 or Node 20 depending on if it's running backend.yml or frontend.yml.

Phase 2: Docker Hub Image Compilation

The runner logs into Docker Hub using encrypted Vault Secrets natively mounted to its environment variables. It recursively loops through each service (e.g. /auth-service) and builds the local Dockerfile.

- name: Build & Push Gateway
  run: |
    docker build -t udayasri/skillsync-gateway ./api-gateway
    docker push udayasri/skillsync-gateway

Phase 3: Secure SSH Penetration

Once all images are rebuilt and staged in Docker Hub, GitHub leverages the appleboy/ssh-action script to forcefully break into our remote AWS EC2 instance using our injected `.pem` key.

Phase 4: Remote EC2 Execution

Once inside the EC2 Box, the action executes a direct shell script overriding the live server state without manual human entry.

script: |
  cd ~/SkillSync
  docker compose pull
  docker compose up -d --remove-orphans
  docker system prune -f

2. The Live Firewall (Nginx Edge Route)

The EC2 Firewall physically blocks all ports except 80 and 443. Nginx operates as the sole guardian at the perimeter of the instance mapping inbound web strings to hidden internal ports.

Cryptographic Decryption

When a user navigates to https://skillsync.udayasri.dev, the 443 request hits Nginx. Nginx utilizes Cerbot's LetsEncrypt keys to decrypt the SSL cipher. It then begins to read the raw HTTP string.

The Proxy Rule Engine

If the URL specifically matches /api/ or /ws/, Nginx rips it from the public internet and drops it into the internal Docker network straight to the Spring API Gateway (Port 8080).

location /api/ {
  proxy_pass http://localhost:8080/;
  proxy_set_header Host $host;
  proxy_set_header X-Real-IP $remote_addr;
}

If the URL does NOT match /api/, Nginx drops it at the Frontend container (Port 5173), loading the Vite React script.

3. The Container Ecosystem

Detailed view of how Docker Compose structures the OS virtualization layer natively on EC2.

The "skillsync-net" Bridge

We do not map microservices to localized IPs (127.x.x.x). We declare a Docker Bridge Network. This means the auth-service container can literally reach the user-service container by executing a REST call to http://user-service:8082 without needing an IP address.

Volume Persistence

Docker containers are inherently ephemeral. If they crash, their hard drives wipe permanently. To ensure database safety, we bind Host paths to Container paths.

volumes:
  - ./data/postgres:/var/lib/postgresql/data

This command instructs the container to literally save the Postgres tables directly onto the AWS EBS volume so it survives reboots flawlessly.
